Likely culprits in order:
1. The .gha that loaded is the OLD one (v0.1.7) — `_PluginManager` in

   v2 component's Debug output

## Notes on known limits (don't be surprised)

- **Execute mode dynamic command tools** (e.g. `gh_skill_command_build_default_hill`
  as a first-class MCP tool) is NOT in v0.2.0 — Skills expose their
  workflow via the `commands:` documentation + `allow_tools:` allowlist.
  Dynamic registration is a v0.2.x deliverable.
- **gh_introspect_definition** handles `.ghx` (raw XML) and modern zip-archive
  `.gh`. Older single-stream binary `.gh` files return a "needs GH runtime"
  stub. Sufficient for any reference file authored in Grasshopper 8+.
- **Highlight colors are uniform teal**. Differentiating added (teal),
  modified (orange), removed (red dashed) requires upgrading
  `RecordTurnChange(Guid)` to take a `ChangeKind` enum — intentional next
  bite, v0.2.y.
- **Mid-session Scenario flip handles correctly** via the Coach prompt
  being self-gated. The prompt sits in MCP `instructions` (set once),
  but it only kicks in when the AI sees `scenario: coach` from
  `get_capabilities`. So flipping Author → Coach mid-chat works.
